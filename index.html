Teoria Prática Links

1. Parte Teórica

Classe Abstrata: Utilizamos Produto como classe abstrata para representar um produto genérico, com subclasses concretas como ProdutoFisico e ProdutoDigital.

Interface: Criamos a interface Descontavel para representar a habilidade de aplicar descontos. A classe Carrinho implementa essa interface.

Polimorfismo: Com a abstração, é possível tratar ProdutoFisico e ProdutoDigital como Produto, facilitando o uso em listas.

Exceções: Foram criadas duas exceções personalizadas: SaldoInsuficienteException e ProdutoInexistenteException.

2. Parte Prática

Classe Abstrata

public abstract class Produto {
  private String nome;
  private double preco;
  // ...
}

Interface

public interface Descontavel {
  double aplicarDesconto(double valor);
}

Exemplo de Uso

Produto livro = new ProdutoFisico("Livro", 50);
carrinho.adicionarProduto(livro);

try {
  carrinho.finalizarCompra(30);
} catch (SaldoInsuficienteException e) {
  System.out.println("Erro: " + e.getMessage());
}
